# aigit 项目方案

## 1. 项目概述

### 1.1 项目名称
**aigit** - AI-powered Git 辅助 CLI 工具

### 1.2 项目定位
- 一个基于 AI 的 Git 工作流增强工具
- 全局 CLI 工具，可在任意 Git 仓库中使用
- 开源发布，面向开发者社区

### 1.3 核心价值
自动化日常 Git 操作流程，通过 AI 生成智能 commit message，并在关键节点设置门控确认，确保操作安全可控。

---

## 2. 技术选型

| 类别 | 选择 | 说明 |
|------|------|------|
| 运行时 | Bun | 快速启动，原生 TypeScript 支持 |
| 语言 | TypeScript | 类型安全，良好的开发体验 |
| 交互方式 | CLI + 简单动画 | 使用 spinner、进度条等增强体验 |
| 配置存储 | 全局配置文件 | `~/.aigit/config.json` |
| AI 模型 | 多模型支持 | 支持 Claude API、Ollama 本地模型等 |

## 3. 核心功能

### 3.1 智能 Commit Message 生成

**输出格式：**
```
<type>: <description>

- 1. xxx
- 2. xxx
- 3. xxx
```

- 分析 `git diff --staged` 内容
- 调用 AI 模型生成结构化的 commit message
- 支持中文/英文输出（可配置）
- 内置过滤规则（忽略空格、注释、构建产物等）
- 列表项最多 7 条，自动合并相似变更
- 优先级：删除 > 新增 > 变更
- 支持用户编辑确认后提交

### 3.2 自动化工作流命令
**命令示例：`aigit sync` 或 `aigit push`**

执行流程如下：

```
┌─────────────────────────────────────────────────────────────┐
│  Step 0: git add .                                          │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 检查远端是否存在当前分支                            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 如果存在且本地落后远程：                             │   │
│  │   → 门控确认：是否下载远程更新到本地？(y/n)          │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 2: 检查本地 master 与远端同步状态                      │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 如果本地 master 落后远端 master：                      │   │
│  │   → 门控确认：是否拉取远程 master？(y/n)             │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 3: 检查当前分支是否包含最新 master                     │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 如果当前分支不包含最新 master：                        │   │
│  │   → 门控确认：是否合并 master 到当前分支？(y/n)        │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 4: 生成并确认 commit message                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 调用 AI 分析变更，生成 commit message                  │   │
│  │   → 门控确认：显示 message，用户可编辑或确认         │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 5: 执行 git commit && git push                        │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 门控确认机制
- 所有关键操作前暂停，等待用户确认（y/n）
- 显示清晰的操作说明和潜在影响
- 支持 `--yes` 或 `-y` 标志跳过非危险确认

## 4. 命令设计

仅保留两个核心命令：

| 命令 | 说明 |
|------|------|
| `aigit commit` | 执行 Step 0-4：`git add .` → 门控检查 → 生成 commit message → 用户确认 → `git commit` |
| `aigit push` | 执行 Step 0-5：`git add .` → 完整门控检查 → 生成 commit message → 用户确认 → `git commit` → `git push` |

### 全局标志
```bash
aigit commit -y        # 跳过非危险确认
aigit push -y          # 跳过非危险确认
aigit push --dry-run   # 预览执行流程，不实际执行
aigit push --verbose   # 显示详细日志
```

## 5. 配置系统

### 5.1 配置文件位置
```
~/.aigit/config.json
```

### 5.2 配置文件结构
```json
{
  "ai": {
    "provider": "openai",
    "api_key": "sk-xxx",
    "baseURL": "https://api.openai.com/v1",
    "model": "gpt-4o-mini"
  },
  "git": {
    "defaultBranch": "main"
  },
  "commit": {
    "language": "zh"  // zh 或 en
  },
  "filter": {
    "ignoreWhitespace": true,
    "ignoreComments": true,
    "ignoreBuildArtifacts": true,
    "mergeCancelingOps": true,
    "maxListItems": 7
  },
  "confirm": {
    "level": "interactive"
  },
  "templates": {
    "commit": []
  }
}
```

## 8. 风险与注意事项

| 风险 | 应对措施 |
|------|----------|
| Git 操作可能导致数据丢失 | 所有危险操作必须门控确认；提供 `--dry-run` 预览 |
| API Key 泄露风险 | 配置文件设置合理权限；不考虑版本控制 |
| AI 生成的 commit message 不准确 | 必须经过用户确认才能提交 |
| 不同 Git 工作流差异 | 支持配置默认分支名（main/master） |
